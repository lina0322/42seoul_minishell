## 설명
### dhyeon과 llim만의 shell입니다.🎉

- 새로운 명령어를 입력할 수 있는 **프롬프트**를 보여줍니다.
- Bash처럼 **(PATH 변수나 상대, 절대 경로를 활용해)** 올바른 **실행파일을 찾고 실행**할 수 있습니다.
- 다음 내장 기능들을 Bash처럼 실행할 수 있도록 구현해보았습니다.
    - ’-n’ 옵션을 활용할 수 있는 **echo**
    - 오직 상대, 절대 경로만 활용하는 **cd**
    - 아무 옵션 없이 **pwd**
    - 아무 옵션 없이 **export**
    - 아무 옵션 없이 **unset**
    - 아무 옵션, 인자 없이 **env**
    - 아무 옵션 없이 **exit**
- Bash처럼 명령문 안의 **;** 는 명령어를 구분할 수 있습니다.
- **여러줄의 명령어를 제외**하곤 **‘** 나 **“** 가 Bash처럼 작동하게 구현하였습니다.
- 파일 디스크립터 집계를 제외하고 리다이렉션 **< > ">>"** 가 Bash처럼 작동합니다.
- 파이프 **|** 가 Bash처럼 작동합니다.
- **환경변수** ($ 다음 문자로 이뤄지는 형식)가 Bash처럼 작동합니다.
- **$?** 가 Bash처럼 작동합니다.
- **'Ctrl-C, ctrl-D, ctrl - \'** 가 Bash와 동일한 결과를 보여줍니다.

## 컨벤션
### C
- Norm v3.를 따른다
- 함수는 동사, 변수는 명사, 전역변수는 대문자로 한다.

### 커밋 규칙
- 가능한 작은 단위로 커밋 하기
- 실행 가능한 상태에서 커밋 하기

### 커밋 메시지 규칙
- 제목의 Type은 영문으로 작성
- Type #issueNumber - 내용 ex) Add #2 - UML 작성 / (이슈번호 없는경우) Refactor - 전반적인 코드 수정
- 타입의 첫글자는 대문자로 작성
- Type 규칙  
    - Add: 파일이나 폴더 추가  
    - Feat: 새로운 코드나 기능 추가  
    - Remove: 필요 코드 제거  
    - Style: 기능이나 로직 변경 없이, 오탈자 등 변경  
    - Refactor: 변수 이름을 바꾸거나, 코드 수정  
    - Docs: 문서 수정 및 추가  
    - Fix: bug 수정(사용자 입장에서 느껴지는 것들)  
    - Chore: 기타 잡 업무  
    - 기타 등등
